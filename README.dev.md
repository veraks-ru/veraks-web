# Локальный запуск кластера

Фронтенд (Next.js) + бэкенд (FastAPI) + Postgres + Redis + мок ЕСИА.

## TL;DR

```bash
# 1. Слой данных (Docker)
docker compose up -d                      # postgres + redis

# 2. Бэкенд + мок ЕСИА (на хосте, из .venv бэкенда)
cd ../backend
python -m venv .venv && . .venv/bin/activate && pip install -e .   # один раз
set -a && . ./.env && set +a              # переменные окружения (см. .env)
alembic upgrade 0009_create_resolutions_disputes   # миграции (billing пропущен)
python seed.py                            # демо-данные + скоринг
uvicorn app.main:app --port 8000 --reload &

cd ../web/dev/mock-esia
../../../backend/.venv/bin/uvicorn app:app --port 9000 &

# 3. Фронтенд
cd ../../                                  # web/
npm install && npm run dev                 # http://localhost:3000
```

Открыть http://localhost:3000.

## Почему гибрид (данные в Docker, приложение на хосте)

В песочнице недоступен Docker Hub (registry EOF), поэтому образ Python не
скачать. Postgres/Redis берутся из локально доступных образов. Когда реестр
доступен, весь кластер поднимается в Docker:

```bash
docker compose --profile full up --build
```

## Вход (мок ЕСИА)

«Войти через Госуслуги» ведёт на мок-страницу выбора учётной записи
(http://localhost:9000/authorize). Доступны засеянные граждане:

- **kalibr** — заходит в аккаунт @kalibr (роль editor, есть история);
- **mediana**, **baseline** — другие засеянные профили;
- «Новый гражданин» — создаёт свежий аккаунт.

Уникальность «1 человек = 1 аккаунт» — по хэшу СНИЛС; повторный вход тем же
гражданином попадает в тот же аккаунт.

## Что подключено к реальному API

- Лента событий, карточка/экран прогноза, фиксация прогноза (PUT).
- Авторизация ЕСИА (redirect → callback → cookie-сессия), `/auth/me`, выход.
- Лидерборды (global/категории/сезон) с резолвингом имён по `user_id`.
- Профиль: калибровка, ранг, по-категориям и история из своих прогнозов.
- Разрешённое событие + шеринговая карточка.

Базовый URL бэкенда — `NEXT_PUBLIC_API_BASE` (см. `.env.local`,
по умолчанию `http://localhost:8000`).

## Известные ограничения локалки

- Миграция `0010_billing` падает на asyncpg (enum-cast при сидинге счётов) —
  биллинг не нужен для 7 экранов, поэтому мигрируем до `0009`.
- Сводка толпы скрыта бэкендом до закрытия приёма (строже, чем по-юзерно) —
  на открытых событиях консенсус показывается после закрытия.
- Сезонная лига пуста: демо-события не привязаны к сезону.
